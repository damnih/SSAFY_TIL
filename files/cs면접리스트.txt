3. LAN과 WAN의 차이점은 무엇인가요?
모범답안:
LAN(Local Area Network)은 집, 학교, 사무실 등 좁은 지역 내에서 사용하는 네트워크입니다. WAN(Wide Area Network)은 지리적으로 넓은 지역을 연결하는 네트워크로, 대표적으로 인터넷이 있습니다. LAN은 자체 구축이 가능하지만, WAN은 ISP 등 외부 통신사업자를 통해 연결해야 합니다.

4. MAC 주소란 무엇인가요?
모범답안:
MAC(Media Access Control) 주소는 네트워크 장비(NIC)에 할당된 고유한 식별자입니다. 48비트(6바이트)로 구성되며, 일반적으로 16진수 형식으로 표현됩니다. 네트워크 내 장치들을 식별할 때 사용되며, 물리 계층과 데이터 링크 계층에서 중요합니다.

5. IP 주소와 MAC 주소의 차이점은 무엇인가요?
모범답안:
MAC 주소는 장비 고유의 물리적 주소이고, IP 주소는 네트워크 상에서 장비를 논리적으로 식별하는 주소입니다. MAC 주소는 변경이 어려운 반면, IP 주소는 네트워크 환경에 따라 변경될 수 있습니다. 일반적으로 데이터 전송 시 ARP를 통해 IP 주소를 MAC 주소로 변환하여 통신합니다.

6. 허브(Hub)와 스위치(Switch)의 차이점은 무엇인가요?
모범답안:
허브는 들어온 데이터를 모든 포트에 무차별적으로 전송하며, 스위치는 MAC 주소를 기반으로 목적지 포트에만 데이터를 전송합니다. 허브는 브로드캐스트 도메인을 분리하지 못하고 충돌 도메인을 공유하지만, 스위치는 각각의 포트가 독립된 충돌 도메인을 형성해 더 효율적입니다.

7. 라우터(Router)의 역할은 무엇인가요?
모범답안:
라우터는 서로 다른 네트워크 간의 데이터를 전달하는 장치입니다. IP 주소를 기반으로 목적지 경로를 찾아 패킷을 전달하며, 라우팅 테이블을 사용해 최적의 경로를 선택합니다. NAT, DHCP, 방화벽 기능도 함께 제공하는 경우가 많습니다.

8. 스위치는 어떤 계층(layer)에서 동작하나요?
모범답안:
스위치는 OSI 7계층 중 2계층인 데이터 링크 계층에서 동작합니다. MAC 주소를 기반으로 패킷을 포워딩하며, 일부 스위치는 3계층 기능(IP 기반 라우팅)도 제공하는 L3 스위치입니다.

9. IP 주소를 분류(Class A, B, C 등)할 수 있나요?
모범답안:
네, IP 주소는 주소 범위에 따라 Class A, B, C 등으로 나뉩니다.

Class A: 1.0.0.0 ~ 126.255.255.255 (대규모 네트워크)

Class B: 128.0.0.0 ~ 191.255.255.255 (중규모)

Class C: 192.0.0.0 ~ 223.255.255.255 (소규모)
현재는 CIDR 방식으로 서브넷 마스크를 유연하게 사용하기 때문에 클래스 개념은 거의 사용되지 않지만, 기본 개념은 여전히 중요합니다.

10. Ping이란 무엇이고 어떤 상황에 사용하나요?
모범답안:
Ping은 네트워크 연결 상태를 확인하는 명령어로, ICMP 프로토콜을 이용합니다. 목적지에 패킷을 보내고 응답 시간을 확인해 해당 장비가 살아있는지 확인할 수 있습니다. 트러블슈팅 시 연결 문제나 지연 시간 확인 등에 주로 사용됩니다.

1. 네트워크 기초 예상 질문 & 모범 답안 (퀴즈 / 모의면접 형태)

Q1. 이더넷(Ethernet)이란 무엇인가요?A1. 다양한 통신 매체의 규격, 송수신되는 프레임의 형태 및 송수신 방식이 정의된 네트워크 기술입니다. 유선 LAN 환경에서 가장 대중적으로 사용되며, IEEE 802.3 표준을 따릅니다.

Q2. IEEE 802.3 표준의 의미는 무엇인가요?A2. 이더넷 관련 표준들의 모음으로, 이더넷의 프레임 구조, 전송 방식, 케이블 종류 등을 정의한 IEEE 전문가 그룹의 표준입니다.

Q3. 이더넷 프레임 구조를 설명해 주세요.A3. 프리엠블 → 수신지 MAC → 송신지 MAC → 타입/길이 → 데이터 → 패딩(필요시) → FCS로 구성됩니다. FCS는 오류 검출을 위한 CRC 값이 들어있습니다.

Q4. MAC 주소란 무엇인가요?A4. 네트워크 인터페이스에 부여된 고유 식별자로, 데이터 링크 계층에서 송수신지를 식별하는 데 사용됩니다.

Q5. NIC(Network Interface Card)의 역할은?A5. 호스트를 네트워크에 연결해주며, 데이터 프레임을 물리 신호로 변환하거나 반대로 신호를 데이터 프레임으로 바꿔주는 역할을 합니다.

Q6. 트위스티드 페어 케이블의 주요 특징은?A6. 구리선 2가닥이 꼬여 있는 구조로, 노이즈 방지를 위해 차폐(Shielding)된 형태도 있습니다. 가장 많이 쓰이는 유선 케이블이며, Cat5, Cat6 등 카테고리별로 성능이 구분됩니다.

Q7. 광섬유 케이블과 트위스티드 페어의 차이점은?A7. 광섬유는 빛을 이용해 데이터를 전송하며, 속도와 거리가 우수하지만 비용이 높습니다. 트위스티드 페어는 전기 신호를 사용하며 근거리 통신에 적합합니다.

Q8. 허브(Hub)의 동작 원리는 무엇인가요?A8. 물리 계층 장비로, 수신한 신호를 모든 포트로 전달합니다. 주소 개념이 없기 때문에 충돌이 자주 발생하며, 반이중 통신만 지원합니다.

Q9. 스위치(Switch)의 차별점은 무엇인가요?A9. 데이터 링크 계층 장비로, MAC 주소를 학습하여 특정 포트로만 데이터를 전달합니다. 전이중 통신을 지원하며 충돌 도메인을 나눌 수 있습니다.

Q10. VLAN이란 무엇인가요?A10. Virtual LAN으로, 하나의 스위치에서 논리적으로 네트워크를 분리할 수 있습니다. 포트 기반 VLAN이 일반적이며, 보안과 트래픽 관리에 유리합니다.
